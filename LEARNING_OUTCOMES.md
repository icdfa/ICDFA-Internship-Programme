# ICDFA Cybersecurity Internship Programme -- Programme Learning Outcomes

**Version:** 1.0  
**Effective Date:** April 2026

---

## Overview

This document outlines the **Programme Learning Outcomes (PLOs)** for the ICDFA Cybersecurity Internship Programme (CIP). These outcomes are aligned with international frameworks including the **NICE Cybersecurity Workforce Framework** and **SFIA (Skills Framework for the Information Age)**.

---

## Track A: Ethical Hacking -- Learning Outcomes

### Upon completion of Track A, interns will be able to:

#### **Knowledge Domain**

1. **Understand Linux Systems Administration**
   - Demonstrate proficiency in Linux command-line operations, file management, and system configuration
   - Apply Linux security best practices in system hardening and user management
   - **NICE Framework:** IT Security Analysis (2210)
   - **SFIA Level:** 3 (Apply)

2. **Understand Network Security Fundamentals**
   - Explain network protocols, traffic analysis, and packet inspection techniques
   - Identify network vulnerabilities and security threats
   - **NICE Framework:** Network Security Analysis (3710)
   - **SFIA Level:** 3 (Apply)

3. **Master Ethical Hacking Methodologies**
   - Conduct reconnaissance, scanning, enumeration, and exploitation in controlled environments
   - Apply penetration testing frameworks and methodologies (e.g., PTES, OWASP)
   - Understand the legal and ethical implications of security testing
   - **NICE Framework:** Penetration Testing (3720)
   - **SFIA Level:** 4 (Enable)

4. **Understand Critical Infrastructure Security**
   - Identify ICS/SCADA systems, PLCs, and industrial protocols
   - Recognize threats specific to critical infrastructure environments
   - Apply defensive measures for industrial control systems
   - **NICE Framework:** Critical Infrastructure Protection (2340)
   - **SFIA Level:** 3 (Apply)

5. **Understand Security Operations**
   - Interpret security logs, alerts, and indicators of compromise
   - Respond to security incidents in a SOC environment
   - Utilize SIEM tools and security monitoring platforms
   - **NICE Framework:** Security Operations Center (SOC) Analyst (3110)
   - **SFIA Level:** 3 (Apply)

6. **Master Applied Security Engineering**
   - Design and implement security controls and countermeasures
   - Configure firewalls, VPNs, and access control mechanisms
   - Understand cryptography, authentication, and authorization systems
   - **NICE Framework:** Security Architecture (2160)
   - **SFIA Level:** 4 (Enable)

7. **Understand Cloud Security**
   - Identify cloud deployment models and security risks
   - Apply cloud security best practices for major platforms (AWS, Azure, GCP)
   - Implement cloud security automation and DevSecOps principles
   - **NICE Framework:** Cloud Security (3050)
   - **SFIA Level:** 3 (Apply)

#### **Skills Domain**

8. **Hands-On Lab Proficiency**
   - Successfully complete 148+ hands-on labs across all courses
   - Demonstrate practical application of security concepts in virtual environments
   - **Assessment:** Lab completion rate ≥ 90%

9. **Report Writing & Documentation**
   - Produce clear, professional security assessment reports
   - Document findings, recommendations, and remediation steps
   - **Assessment:** Capstone report evaluation

10. **Communication & Collaboration**
    - Communicate security concepts to both technical and non-technical audiences
    - Collaborate effectively with peers and instructors in a team environment
    - **Assessment:** Group project participation and peer feedback

#### **Competency Domain**

11. **Critical Thinking & Problem-Solving**
    - Analyze complex security scenarios and develop appropriate solutions
    - Apply troubleshooting methodologies to resolve security issues
    - **Assessment:** Lab scenario completion, capstone project

12. **Professional Conduct & Ethics**
    - Adhere to ethical hacking principles and legal requirements
    - Maintain confidentiality and professional standards
    - **Assessment:** Code of conduct compliance, peer reviews

## Track B: Digital Forensics -- Learning Outcomes

### Upon completion of Track B, interns will be able to:

#### **Knowledge Domain**

1. **Understand Foundational Computing for Forensics**
   - Demonstrate proficiency in number systems, PC architecture, and command-line workflows relevant to forensic work
   - Apply both Windows and Linux command-line skills during evidence discovery and validation
   - **NICE Framework:** Digital Forensics (3130)
   - **SFIA Level:** 3 (Apply)

2. **Master Core Digital Forensics Methodologies**
   - Acquire, preserve, and examine digital evidence using accepted forensic procedures
   - Apply imaging, metadata analysis, evidence search, data carving, and steganography techniques
   - **NICE Framework:** Digital Forensics (3130)
   - **SFIA Level:** 4 (Enable)

3. **Conduct Network Forensic Investigations**
   - Analyze packets, sessions, and protocol artefacts to identify suspicious or malicious activity
   - Reconstruct events from HTTP, SMTP, DNS, wireless, and firewall evidence
   - **NICE Framework:** Network Operations Specialist / Forensic Analysis (3140)
   - **SFIA Level:** 4 (Enable)

4. **Interpret Windows and Endpoint Artefacts**
   - Examine registry, event, browser, file system, and timeline artefacts from structured case datasets
   - Correlate endpoint evidence to reconstruct user and attacker activity
   - **NICE Framework:** Forensic Analysis (3140)
   - **SFIA Level:** 4 (Enable)

5. **Apply Case-Based Investigative Techniques**
   - Conduct focused forensic investigations involving email, memory, steganography, network traces, and historical attack scenarios
   - Evaluate evidential relevance, reliability, and limitations across different case types
   - **NICE Framework:** Incident Response (3120)
   - **SFIA Level:** 4 (Enable)

6. **Analyze Mobile and IoT Evidence**
   - Extract and interpret evidence from smart-home, voice assistant, Android, iPhone, and drone datasets
   - Explain the role of chip-off workflows, SQLite analysis, and app artefacts in mobile/IoT investigations
   - **NICE Framework:** Digital Forensics (3130)
   - **SFIA Level:** 4 (Enable)

7. **Use AI-Assisted Forensic Analysis Responsibly**
   - Apply AI tools to evidence summarisation, suspect profiling, and forensic intelligence analysis while preserving validation and human review
   - Produce defensible findings that distinguish between machine assistance and examiner judgement
   - **NICE Framework:** Forensic Analysis (3140)
   - **SFIA Level:** 4 (Enable)

#### **Skills Domain**

8. **Hands-On Lab Proficiency**
   - Successfully complete 90+ labs and guided investigative units across the redesigned Track B curriculum
   - Demonstrate practical evidence acquisition, analysis, and reporting in case-driven environments
   - **Assessment:** Lab completion rate ≥ 90%

9. **Forensic Report Writing**
   - Produce clear, professional forensic notes, timelines, and final reports
   - Document acquisition methods, evidential reasoning, limitations, and conclusions in a legally aware manner
   - Present findings suitable for internal investigations or formal review
   - **Assessment:** Capstone report evaluation

10. **Communication & Collaboration**
    - Communicate forensic findings to both technical and non-technical audiences
    - Collaborate effectively with peers and instructors in case review and evidence interpretation exercises
    - **Assessment:** Group project participation and peer feedback

#### **Competency Domain**

11. **Critical Thinking & Analytical Skills**
    - Analyze complex forensic scenarios and develop defensible investigative strategies
    - Apply structured reasoning to evidence correlation, timeline construction, and hypothesis testing
    - **Assessment:** Lab scenario completion, capstone project

12. **Professional Conduct & Ethics**
    - Adhere to forensic investigation standards, chain-of-custody practices, and legal/ethical requirements
    - Maintain evidence integrity, transparency of method, and professional standards throughout all investigations
    - **Assessment:** Code of conduct compliance, peer reviews

## Cross-Track Competencies

All interns, regardless of track, will develop:

- **Technical Competency:** Proficiency with industry-standard security and forensic tools
- **Analytical Competency:** Ability to analyze complex security scenarios and data
- **Communication Competency:** Clear articulation of technical findings to diverse audiences
- **Professional Competency:** Adherence to ethical standards and professional conduct
- **Collaborative Competency:** Effective teamwork and peer support

## Alignment with International Frameworks

### NICE Cybersecurity Workforce Framework

The CIP curriculum aligns with the following NICE work roles:

**Track A (Ethical Hacking):**
- Penetration Tester (3720)
- Security Operations Center (SOC) Analyst (3110)
- Network Security Analyst (3710)

**Track B (Digital Forensics):**
- Digital Forensics Analyst (3130)
- Incident Responder (3120)
- Forensic Analyst (3140)

### SFIA (Skills Framework for the Information Age)

The CIP curriculum aligns with SFIA levels 3-4 across multiple skill categories:

- **Security Architecture:** Level 4 (Enable)
- **Network Security:** Level 3 (Apply)
- **Incident Management:** Level 3 (Apply)
- **Digital Forensics:** Level 4 (Enable)
